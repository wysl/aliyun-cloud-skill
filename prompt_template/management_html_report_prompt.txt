使用 aliyun-cloud-skill skill，为以下对象生成一份“阿里云资源使用情况管理层简版 HTML 报告”：

- 公司/客户名称：<公司名>
- 账号名称：<account>
- 报告周期：<YYYY-MM 或 近7天/近30天>
- 输出文件：<文件名>.html

目标：
生成一份适合管理层阅读的 HTML 报告，突出资源概况、成本、风险、重点异常和关注项。
不要堆砌原始命令输出，不要写过多技术细节，但必须保证内容完整。

执行规则：
1. 先列出报告章节清单。
2. 再逐项调用 aliyun-cloud-skill skill 查询。
3. 任何章节都不能省略。
4. 某项查不到时，必须保留章节并写“未获取到 + 原因”。
5. 所有查询完成后，先输出数据完整性检查表，再生成最终 HTML。
6. 不允许虚构数据。

报告章节至少包括：
1. 报告摘要
2. 账户概览
3. 成本与余额概览
4. 核心资源规模概览
5. 重点资源使用率概览
6. 网络与访问层概览
7. 存储与分发概览
8. 域名与证书风险
9. 风险与关注项
10. 数据完整性检查表
11. 附录：执行命令清单

各章节要求：

- 报告摘要
  - 用 5-10 条要点总结：
    - 当前资源规模
    - 本期账单/余额情况
    - 是否存在高使用率资源
    - 是否存在证书即将到期
    - 是否存在资源包即将耗尽/到期
    - 是否存在数据缺失

- 账户概览
  - 账号名称
  - 覆盖区域
  - 本次实际可获取的数据源（阿里云 API / Grafana / SLS）

- 成本与余额概览
  - account-balance
  - bill-summary
  - 用简明表格展示
  - 给出一句业务化说明

- 核心资源规模概览
  - ECS 数量
  - RDS 数量
  - PolarDB 数量
  - ALB 数量
  - OSS Bucket 数量
  - CDN 域名数量
  - SLS Project 数量
  - 适合做成总览卡片或汇总表

- 重点资源使用率概览
  - ECS 使用率
  - RDS 使用率
  - PolarDB 使用率
  - ALB 使用率
  - 只保留高层可读指标：
    - 高负载资源数量
    - Top 风险资源
    - 是否需要扩容/关注

- 网络与访问层概览
  - VPC / VSwitch / RouteTable 总览
  - 安全组数量与重点规则情况
  - ALB ACL 配置情况（如可获取）

- 存储与分发概览
  - OSS：存储量、对象数、流量、请求量
  - CDN：流量、带宽、回源流量、命中率
  - 用管理层易懂的描述，不展开技术实现细节

- 域名与证书风险
  - domain-list
  - ssl-summary
  - ssl-expiring
  - 重点显示即将到期项

- 风险与关注项
  - 高使用率资源
  - 即将到期证书
  - 即将到期/低余量资源包
  - 数据缺失项
  - 监控缺失项
  - 每项风险后附一句建议

- 数据完整性检查表
  - 章节名
  - 是否成功
  - 是否有数据
  - 失败/缺失原因

- 附录：执行命令清单
  - 按章节列出实际执行命令

输出顺序必须严格为：
A. 章节清单
B. 逐项查询过程
C. 数据完整性检查表
D. 最终 HTML

最终 HTML 风格要求：
- 偏汇报风格
- 开头先给“执行摘要”
- 多用总览表和风险摘要框
- 少放原始 JSON
- 每章必须保留
- 对缺失数据用明显提示框标注“未获取到 + 原因”
