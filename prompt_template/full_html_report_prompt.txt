使用 aliyun-cloud-monitor skill，为以下对象生成一份“阿里云资源使用情况 HTML 报告”：

- 公司/客户名称：<公司名>
- 账号名称：<account>
- 报告周期：<YYYY-MM 或 近7天/近30天>
- 输出文件：<文件名>.html

目标：
生成一份完整、可直接交付的 HTML 报告，覆盖当前 skill 能提供的主要只读查询/监控内容。不要只给摘要，不要只列命令结果，要整理成结构化报告。

执行要求：
1. 必须先列出“本次报告章节清单”。
2. 然后按章节逐项调用 aliyun-cloud-monitor skill 查询数据。
3. 每个章节都要明确：
   - 使用的命令
   - 查询是否成功
   - 是否有数据
   - 如果失败/无数据，失败原因是什么
4. 任何章节都不允许被静默省略。
5. 如果某项由于权限不足、账号未开通服务、区域无资源、Grafana 未配置、Prometheus 无数据、命令报错等原因无法获取：
   - 必须保留该章节
   - 必须写“未获取到”
   - 必须写明具体原因
6. 查询全部完成后，先输出“数据完整性检查表”，再生成最终 HTML。
7. 最终 HTML 的章节数必须与一开始列出的章节清单一致。
8. 不允许虚构数据；没有就明确写没有。
9. 对数值类信息优先使用表格展示。
10. 对需要 Grafana 的使用率类数据，如果拿不到，必须保留章节并注明“Grafana 配置缺失或查询失败”。

报告必须至少包含以下章节：
1. 报告说明
2. 账户概览
3. 账单与余额
4. ECS 资源概览
5. ECS 使用率监控
6. RDS 资源概览
7. RDS 使用率监控
8. PolarDB 资源概览
9. PolarDB 使用率监控
10. ALB 资源概览
11. ALB 使用率监控
12. VPC / VSwitch / 路由表概览
13. 安全组概览
14. 域名与 SSL 证书概览
15. OSS 使用情况
16. CDN 使用情况
17. SLS 概览
18. 风险与关注项
19. 数据完整性检查表
20. 附录：本次实际执行的命令清单

各章节建议查询内容如下：

- 账户概览
  - 账号名称
  - 区域范围
  - 本次可用的数据源类型（阿里云 API / Grafana / SLS 等）

- 账单与余额
  - account-balance
  - bill-summary

- ECS 资源概览
  - ecs-list
  - 按区域统计实例数、运行状态、规格、IP、到期时间（如果有）

- ECS 使用率监控
  - ecs-usage
  - CPU / 内存 / 磁盘
  - 标出高使用率实例

- RDS 资源概览
  - rds-list
  - rds-detail（必要时）
  - rds-list-backups（必要时）

- RDS 使用率监控
  - rds-usage
  - rds-performance（必要时）
  - CPU / 内存 / IOPS

- PolarDB 资源概览
  - polardb-list

- PolarDB 使用率监控
  - polardb-usage

- ALB 资源概览
  - alb-list
  - alb-acl-list
  - alb-listener-acl（必要时）

- ALB 使用率监控
  - alb-usage
  - 带宽 / 活跃连接 / QPS / 错误率

- VPC / VSwitch / 路由表概览
  - vpc-list
  - vswitch-list
  - routetable-list
  - vpc-detail（必要时）
  - vswitch-resources（必要时）

- 安全组概览
  - sg-list
  - sg-rules（必要时抽样或按重点组展开）

- 域名与 SSL 证书概览
  - domain-list
  - ssl-list
  - ssl-expiring
  - ssl-summary

- OSS 使用情况
  - oss-list
  - oss-usage
  - 必须展示：存储量、对象数、流量、请求量
  - 必要时补充 oss-info / oss-recent

- CDN 使用情况
  - cdn-list
  - cdn-traffic
  - cdn-bandwidth
  - cdn-usage
  - 必须展示：流量、带宽、回源流量、命中率

- SLS 概览
  - sls-list-projects
  - sls-list-logstores（必要时）
  - sls-get-index / sls-get-logs / sls-query-ips（必要时）
  - 如果没有使用 SLS，也必须保留章节说明

- 风险与关注项
  - 汇总高使用率资源
  - 即将到期证书
  - 即将到期资源包
  - 配置缺失项
  - 查询失败项
  - 空数据项

输出顺序必须严格遵守：
A. 本次报告章节清单
B. 逐章节查询过程
C. 数据完整性检查表
D. 最终 HTML（完整可落盘版本）

最终 HTML 要求：
- 使用清晰的标题层级（h1/h2/h3）
- 每个章节必须存在
- 表格展示关键资源和关键指标
- 对“未获取到”的章节使用醒目标注
- 在文末附“本次执行命令清单”
- 不要输出与最终报告无关的闲聊内容

如果你准备跳过某个章节，请先停止并说明原因，不允许默默省略。
任何未成功获取的数据都必须以“保留章节 + 标注未获取到 + 写明原因”的方式进入最终 HTML，禁止因为失败而删除章节。
